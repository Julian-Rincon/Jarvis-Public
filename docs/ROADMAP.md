# Roadmap

## Near Term

- continue stabilizing the packaged Windows app
- improve controlled startup and recovery behavior
- expand diagnostics for audio and service failures
- prepare sanitized demo assets for portfolio use

## Mid Term

- isolate more runtime services into independent processes
- improve browser and desktop automation reliability
- deepen Google Workspace workflows
- strengthen memory and profile management

## Longer Term

- move toward a more service-oriented architecture
- add broader home and device orchestration
- improve proactive planning and reminder behavior
- prepare a stronger operator console for sustained daily use

## Non-Goals for the Public Repo

- open-source release of the production code
- exposure of private workstation automation details
- publication of local secrets or personal context files
